## 一、项目基础信息
- **项目全称**：SimDiff_improved 基于扩散模型的时间序列预测方法研究
- **基础模型**：开源条件扩散模型 SimDiff（核心框架、归一化独立性NI机制、DDIM快速采样、Median-of-Means鲁棒点聚合均为原模型已有功能）
- **核心任务**：面向表格型时间序列的**多变量多步前瞻概率预报**，给定长度为`seq_len=96`的多变量历史观测窗口，预测未来`pred_len=24`步的所有变量序列值（支持调整窗口长度，修改`pred_len`需对应重新训练）
- **实验数据集**：weather（小时级气象）、wind（15分钟风电）、ETTh1（小时级电力负荷）、exchange_rate（日度汇率）
- **对比基线**：iTransformer、TimeMixer（共享同一DataLoader与训练评估流程，保证对比公平性）
- **评估指标**：点预测指标（MAE、MSE，同时输出物理尺度与z口径）、概率预测指标（CRPS连续分级概率评分、VAR样本方差）

## 二、**你的唯一两个创新点**（其余均为原模型或工程化实现内容）
1.  **多尺度历史特征融合**：突破仅使用最近细粒度历史序列的局限，在预测边界左侧按日历语义拼接7个日尺度均值token与4个周尺度均值token，将历史条件总长度扩展为`96+11=107`，增强模型对长期周期性模式的捕捉能力
2.  **RMSNorm归一化优化**：采用Pre-RMSNorm替代原模型的LayerNorm归一化方式，提升模型训练的稳定性与收敛速度
- **模型默认配置**：多尺度历史开启、RMSNorm开启、RevIN关闭（RevIN需通过`--revin` CLI参数手动开启）

## 三、核心功能与关键约束
- 本研究**仅采用多变量联合预测模式**（通过`--all_features`参数开启），利用变量间的相关性提升预测精度
- 基线默认采用`BaselineHistTrim`机制：仅使用前96步细粒度历史序列，与使用全长多尺度条件的SimDiff存在信息量不对称（公平对比需添加`--baseline_full_hist`参数）
- 点预测基线的概率指标语义：iTransformer/TimeMixer无多采样能力，其CRPS在δ(点预测)分布下与MAE数值完全一致，VAR恒为0（需在指标表脚注说明）
- 多变量模式下的结果规范：终端指标表为**全通道平均结果**，可视化曲线图仅展示主变量（如风电OT列、气温列）的预测效果（需在图注说明）
- `pred_len`与模型权重强绑定：修改未来预测步长必须重新训练对应结构的模型，可通过`--ckpt_extra_suffix`参数防止权重文件覆盖

## 四、实验设计与成果产出
- **唯一核心消融实验**：多尺度历史×RMSNorm四柱消融实验（通过`--ms_rms_ablation`入口执行），量化两个创新点对模型性能的独立贡献与协同效应
- **自动结果输出**：
  1.  终端ASCII格式的标准化毕设指标表
  2.  多模型MAE/MSE对比柱状图
  3.  历史序列+真实值+多模型预测值叠加曲线图
- **结果存储规则**：wind数据集结果默认存放在项目根目录`wind/`文件夹，其余数据集结果存放在`result/<数据集名称>/`文件夹；消融实验结果图会自动复制到`xiaorong/`文件夹的固定文件名，便于论文引用
- **工程化实现**：默认开启混合精度训练AMP、EMA权重平滑、早停机制；基于验证集噪声损失进行模型选优（基于验证集forecast MAE的选优策略默认关闭，开启会显著增加训练耗时）

### 5.1 与代码仓库表述一致性的重要勘误
- **单变量 vs 多变量**：仓库 **`Config.temperature_only` 默认 True**，不加 **`--all_features`** 时多为**单列目标**（如某数据集上的 OT 或气温列），`input_dim=1`。本文第三节写「**仅采用**多变量联合预测」易被理解为**整个项目只支持多变量**，与实现不符。建议改为二选一，并在答辩时能说清：
  - 若你**毕设实验**统一做「全列联合预测」：可写「**本论文实验章节**统一采用 `--all_features` 多变量设定」；
  - 若需要概括**整个系统能力**：应写「支持单变量默认与 `--all_features` 多变量两种模式」。
- **项目名称**：正文写 **SimDiff_improved** 时，建议**脚注或封面**交代代码仓库/目录实际名称（如 Simdiff_weather），避免查重材料与路径不一致。
- **数据集列举**：除 weather、wind、ETTh1、exchange_rate 外，仓库管线还支持 **ETTm1** 等；若论文只用了其中一部分，列表写「主要包括…」即可。

### 5.2 「创新点」表述的学术分寸（建议与导师确认）
- **多尺度历史**与 **RMSNorm 编码栈** 属于在**开源 SimDiff 框架上的结构/数据侧扩展**；不同院校对「创新点」与「改进工作」认定不同。对外表述可备两套：**（1）工程与创新扩展**；（2）若答辩老师追问文献，可退守为「**针对表格时间序列预报任务的有效改进与系统验证**」。

### 5.3 可在「系统实现」一笔带过、但容易被问到的点
- **SimDiff 主线消融**：除 **`--ms_rms_ablation`** 外，`simdiff_ablation` 仍可能涉及 **NI/MoM** 语义（如 `ni_only` / `mom_only`）；若正文只写 ms_rms，可加一句「概率聚合与归一化独立性沿用原 SimDiff 设计」。
- **基线 TimeMixer**：实现多为**课程/毕设友好型精简版**，建议在论文或脚注说明**与原文/官方实现的差异**，避免被质疑「基线过弱」无章可循。
- **自检验证**：根目录 **`verify_norm_mom.py`** 可做 NI+MoM 一致性自检；可写在「可复现性」或附录一行。
- **权重与数据集**：`wind` / `exchange_rate` 等在**未手写后缀**时，代码里可能对**单变量**与**多变量**使用不同默认 **`--ckpt_extra_suffix`**（如 `_wind` vs `_wind_mv`），避免不同实验互相覆盖——写实验设置时建议**明确写出**每次使用的 checkpoint 全名。
- **可视化与指标**：若 overlay 使用了**仅作图**的竖直锚定（`anchor_forecast_boundary`），**终端 MAE/MSE/CRPS 仍以未平移预测为准**；图注中可点明，避免「图与表是否同一套数」被追问。
- **诚信与配置**：若使用过 **`thesis_gt_peek`**（仅对 SimDiff 曲线与真值混合作图），**不得**与真实评测指标混报；须在方法或脚注中区分「展示参数 λ」与正式指标。

### 5.4 可选补一句「局限与展望」（答辩收尾）
- 早停/选优默认依据**验证集扩散噪声损失**，与**物理尺度 MAE** 不完全一致，可能出现「损失已降而曲线观感一般」——可作为局限或未来工作：**按验证集 forecast MAE 选优（代价是算力）** 或 **多指标联合早停**。

## 六、Normalization Independence（NI）逻辑发掘与「是否有问题」辨析（追加）
--------------------------------------------------------------------------------

本节对应仓库内 **Normalization Independence** 的**实际实现**与常见质疑（含根目录旁注 `bug/3.txt`），便于你答辩时区分「设计本意」「评测口径」「可选消融分支」。

### 6.1 NI 在本仓库中的定义（与 SimDiff 叙事一致）

**核心约定**：同一滑动窗口内，**历史段**与**未来段**各自在时间维上估计 \(\mu,\sigma\)，**互不混用**对方像素/时间步上的原始值来算对方的尺度。

| 对象 | 统计量来源 | 代码位置 |
|------|------------|----------|
| 条件输入 `hist_n` | \(\mu_h,\sigma_h\) 仅由历史张量估计；多尺度时为 **前 `seq_len` 步细粒度段**，避免日/周池化 token 污染尺度 | `utils/independent_normalizer.py` → `normalize_history(..., hist_stats_span=...)`；主流程固定传 `hist_stats_span=int(cfg.seq_len)`，见 `models/simdiff.py` |
| 扩散目标 `fut_n` | \(\mu_f,\sigma_f\) **仅由未来 `pred_len` 步**在时间维上估计 | `normalize_future` |
| 反变换到物理尺度 | \(y = z\cdot\sigma+\mu\)，与上式可逆 | `inverse_transform_future` |

**旧 API 兼容**：`utils/normalizer.py` 将 `normalize_pair` 映射到 `IndependentNormalizer`，且 **`independent_future=False` 已废弃**（会抛错），与 README 中更早版本的「历史 σ 统一未来」叙述不同——**以当前 `independent_normalizer.py` + `simdiff.py` 为准**。

### 6.2 调用链概览（训练 → 评测 → 无真值推理）

1. **训练 `training_loss`**（`models/simdiff.py`）  
   - `hist_n = normalize_history(hist, hist_stats_span=seq_len)` → 条件网络仅见 `hist_n`。  
   - **默认 `simdiff_ablation=="full"` / `"ni_only"`**：`fut_n = normalize_future(future)` —— 扩散在 **未来窗自身的 z-score 空间**拟合噪声。  
   - **`simdiff_ablation=="mom_only"`**：`fut_n = (future - μ_h) / σ_h`，与 NI **不一致**，需**单独 checkpoint**，文档与 `simdiff_checkpoint_filename()` 已标明。

2. **有真值评估 `forecast(hist, future=future)`**  
   - `_future_mu_sig_for_inverse`：若有 `future`，先对该 batch 做 `normalize_future(future)` 取出 \(\mu_f,\sigma_f\)，再用于反变换 —— **与训练目标空间一致**（不是「训练偷看 future 喂进网络」，而是「评价时用什么 \(\mu,\sigma\) 把 z 映回物理尺度」）。

3. **无真值推理**（不传 `future`）  
   - 使用 `make_loaders` → `fit_future_marginal_stats(train_ds)` 写在 `cfg` 上、再 `register_buffer` 的 **`train_future_marginal_mean/std`**（训练集所有 **未来窗口** 聚合的边际 \(\mu,\sigma\)，**不含验证/测试 future**）。

4. **可选非严格 NI**：`ni_inverse_hist_frac > 0` 时，反变换对 \(\mu,\sigma\) 做「未来 vs 历史统计」凸组合；**论文正式对比应保持 0**（`simdiff.py` 文件头注释已强调）。

### 6.3 多尺度与 NI 的耦合点（历史上出过「整体平移」类 bug 的位置）

- 多尺度下 `hist` 长度为 **`seq_len + 11`**（`Config.effective_hist_len()`）。  
- 若误用「整段 Lh」估 \(\mu_h,\sigma_h\)，日/周均值段的数值尺度与小时段不同，会 **污染** \(\mu_h,\sigma_h\)，导致条件分布与目标空间错位；本仓库已通过 **`hist_stats_span=seq_len`** 修复（详见 `docs/DEVELOPMENT_LOG.md` 对应条目）。  
- **要点**：NI 的「独立」指历史/未来 **不交叉取统计**；**hist 内部**仍需要用正确前缀定义「何为历史段的尺度」——这是实现细节，不是 NI 定义矛盾。

### 6.4 你觉得「NI 有问题」时，通常指的是哪几类问题？

下面区分 **误解**、**设计权衡**、**真实现状**，避免一锅粥。

**（A）「训练用了 future 的 \(\mu_f,\sigma_f\)，算不算标签泄漏？」**  
- **网络前向**：去噪器 **只接收 `hist_n`**，没有把 future 序列当作额外输入通道。  
- **损失定义**：目标 `fut_n` 用了 **本窗 future** 的均值/方差，等价于让模型在「**以该窗 future 为参照的坐标系**」里预测形状；窗的全局水平被 \(\mu_f\) 吸收一部分。  
- **学术口径**：这与「把 future 值 leak 进输入」不同，但确实属于 **「训练目标坐标系依赖测试期未知的窗内统计」** —— SimDiff 类 NI 论文意图是 **概率预报在规范化空间**，并与 **有真值时用同窗 \(\mu_f,\sigma_f\) 反变换** 对齐。  
- 根目录 `bug/3.txt` 将此定性为「致命泄漏」并要求改成「全程用 \(\mu_h,\sigma_h\)」—— **那是另一条设计路线**（与本仓库默认 NI **不等价**）；`docs/DEVELOPMENT_LOG.md` 已写明 **未采纳** 该改法，否则会改变训练目标语义。若你需要 **严格开环可部署口径**，应在论文/附录中单列：**仅用边际或历史统计反变换** 的指标，或与 **`mom_only`**（_hist 归一未来_）对照。

**（B）「测试集 MAE 传入 `future` 反变换，算不算作弊？」**  
- 这是 **评测协议选择**：当前实现保证「**z 空间误差 ↔ 物理空间误差**」与训练时 **同一套 \(\mu_f,\sigma_f\)**，指标反映的是 **在该坐标系下的预报质量**。  
- **与真实部署的差异**：部署时常 **没有** 未来窗的 \(\mu_f,\sigma_f\)，只能用 **训练边际** 或别的估计 → **线上误差分布可能与当前测试脚本不一致**。这不是代码自相矛盾，而是 **「论文指标」vs「纯开环指标」** 要分开写清。  
- **建议**：若在答辩中被追问，可答：主表指标与 SimDiff/NI 文献对齐；若导师要求 **零 future 信息**，应汇报 **`forecast(hist)`（不传 future）+ 边际反变换** 的次要表格或脚注。

**（C）「曲线很平、像常数 —— 是不是 NI 坏了？」**  
- `DEVELOPMENT_LOG.md` 已有机制说明：**弱模型在 z 空间贴近 0**，反变换 \(\hat y \approx \mu_f\)（窗内常数偏置），叠加 MoM 平滑，主观上像「水平线」。  
- 这更多是 **拟合不足 + NI 坐标系** 的合力，优先排查 **噪声 loss 与物理 MAE 是否对齐选优**、容量与采样步数等，而不是单独断言 NI 实现写错。

**（D）实现层面可核对的一致性**  
- `verify_norm_mom.py`：**断言 `normalize_history` 不随 future 改变**（NI 对历史的字面含义）；**未来 round-trip 可逆**；MoM ≠ 单次采样。  
- **未覆盖**：「不传 future 的边际反变换 vs 传 future 的窗内反变换」在同一 batch 上的数值差异；跨 split 的分布漂移 —— 若你怀疑部署 gap，可自行加小规模断言或小实验。

### 6.5 小结（写进心里的 checklist）

1. **NI 在本项目 = 历史统计与未来统计分离 + 扩散在未来窗 z 空间**；多尺度必须 **`hist_stats_span=seq_len`**。  
2. **有真值评测传 `future`** 是为了 **反变换与训练目标一致**，不是把 future 喂进网络。  
3. **「泄漏」指控**要分清对象：**输入泄漏**（本实现无）vs **目标坐标系依赖窗内 future 统计**（本实现有，属 SimDiff NI 设计）。  
4. 若论文要写「严格因果可部署」，需 **额外报告** 不传 future、仅用边际（或 `mom_only`/historical affine）的对照，避免口径被质疑。
