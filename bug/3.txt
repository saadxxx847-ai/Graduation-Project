# 实验结果深度分析与改进方案
你的实验结果暴露了**时间序列预测中最致命的标签泄漏问题**，这是导致"指标异常优秀但预测完全失效"的根本原因。以下是系统的问题诊断和可直接落地的改进步骤。

## 一、核心问题诊断（优先级：最高）
### 1. 致命缺陷：Normalization Independence (NI) 存在严重标签泄漏
这是你所有问题的根源，必须**立即修正**，否则整个实验结果无效。

#### 问题本质
你的NI设计违反了时间序列预测的铁律：**任何归一化操作绝对不能使用未来信息**。
- **训练阶段泄漏**：训练时用每个样本自己的**未来序列**计算μ_f和σ_f来归一化目标。模型在训练时就已经知道了未来的整体水平和波动范围，根本不需要学习预测趋势，只需要预测一个围绕0的小波动即可。
- **评测阶段泄漏**：评测时传入future参数（测试集真值），又用该batch的**未来统计量**反变换模型输出。这相当于直接用测试集的真实均值和方差校准预测，使得MAE/MSE看起来异常优秀。

#### 铁证：预测曲线完全退化
看你的预测overlay图：
- 真实值（黑色）在预测段从25快速下降到0附近，呈现明显的下降趋势
- 所有模型（SimDiff、iTransformer、TimeMixer）的预测都是**几乎水平的直线**，完全没有捕捉到任何趋势
- 这说明模型根本没有学到预测能力，只是在预测一个常数（历史均值或未来均值）

### 2. 次要但关键的问题
| 问题 | 影响 |
|------|------|
| 验证早停用噪声损失而非预报MAE | 噪声损失与最终预报性能不完全正相关，可能早停在次优点 |
| 基线对比不公平 | SimDiff用多尺度历史，基线只用细粒度段；归一化方式不同 |
| 扩散模型生成平滑性 | 扩散模型固有特性，容易生成缺乏高频细节和趋势的结果 |
| MoM聚合过度平滑 | 牺牲精度换取鲁棒性，进一步抹平了预测趋势 |

## 二、分阶段改进方案（可直接执行）
### 阶段一：紧急修复标签泄漏（1-2天）
这是最优先的任务，修正后你会看到模型的真实性能。

#### 具体修改步骤
1. **修改`utils/independent_normalizer.py`**
   ```python
   # 删除原来的normalize_future函数，替换为：
   def normalize_future_with_hist_stats(future, hist_mean, hist_std):
       """用历史的均值和方差归一化未来，绝对不使用未来信息"""
       return (future - hist_mean) / (hist_std + 1e-8)

   def inverse_transform_future_with_hist_stats(fut_n, hist_mean, hist_std):
       """用同一个历史统计量反变换"""
       return fut_n * hist_std + hist_mean
   ```

2. **修改`models/simdiff.py`中的训练流程**
   ```python
   # 在training_loss函数中：
   hist_n, hist_mean, hist_std = normalize_history(hist, hist_stats_span=seq_len)
   # 原来的 fut_n = normalize_future(future) → 删除
   fut_n = normalize_future_with_hist_stats(future, hist_mean, hist_std)  # 新增
   ```

3. **修改`models/simdiff.py`中的推理流程**
   ```python
   # 在forecast函数中：
   hist_n, hist_mean, hist_std = normalize_history(hist, hist_stats_span=seq_len)
   # 采样得到归一化轨迹 samples_n
   mom_n = mom_aggregate_normalized(samples_n)
   # 原来的 inverse_transform_future(mom_n, mu_f, sigma_f) → 删除
   mom = inverse_transform_future_with_hist_stats(mom_n, hist_mean, hist_std)  # 新增
   ```

4. **删除所有依赖未来统计量的代码**
   - 删除`make_loaders`中计算`train_future_marginal_mean/std`的代码
   - 删除`forecast`函数中所有使用`future`参数计算统计量的逻辑

#### 预期结果
修正后SimDiff的MAE/MSE会**显著上升**（这是正常的，因为不再有标签泄漏的帮助），但预测曲线会开始捕捉到真实值的趋势。

### 阶段二：解决预测曲线退化问题（3-5天）
修正标签泄漏后，如果预测仍然是水平线，按以下步骤优化：

#### 1. 改进训练目标
- **加入时间差分损失**：鼓励模型预测相邻时间步的差异，而不是绝对值
  ```python
  # 在diffusion.training_losses中加入：
  td_loss = F.mse_loss(pred[:, 1:] - pred[:, :-1], target[:, 1:] - target[:, :-1])
  total_loss = mse_loss + 0.1 * td_loss  # 权重可调整
  ```

- **使用Huber损失代替MSE**：对异常值更鲁棒，避免模型被极端值带偏
  ```python
  loss = F.huber_loss(pred, target, delta=1.0)
  ```

#### 2. 增强去噪网络容量
- 增加Transformer层数：从默认的2层增加到4-6层
- 增加注意力头数：从4头增加到8头
- 增大隐藏层维度：从512增加到768（如果显存允许）

#### 3. 调整扩散采样策略
- 增加DDIM采样步数：从100步增加到200-500步
- 使用更先进的采样器：替换为DPM-Solver++，可以在更少步数下得到更好结果
- 加入引导采样：用历史最后几个值引导采样过程，使生成序列更符合历史趋势

### 阶段三：优化多尺度与基线对比（2-3天）
#### 1. 先验证单尺度性能
- 先运行单尺度的SimDiff实验（`--single_scale_hist`）
- 如果单尺度性能不好，先优化单尺度模型，再考虑多尺度
- 多尺度是锦上添花，不是雪中送炭

#### 2. 改进多尺度融合方式
- 不要简单拼接，使用注意力机制融合不同尺度信息
- 为不同尺度的token设计独立的位置编码
- 使用可学习的池化层代替简单的均值池化

#### 3. 保证基线对比公平性
- 修改`baselines.py`，让iTransformer和TimeMixer也使用**相同的归一化方式**
- 让基线模型也使用**相同的多尺度历史输入**
- 增加单尺度SimDiff与单尺度基线的对比实验

### 阶段四：其他优化（1-2天）
1. **用预报MAE做早停**：每隔5个epoch做一次全流程采样，计算验证集MAE，用MAE做早停
2. **调整聚合方式**：尝试算术均值、加权均值代替MoM，观察效果
3. **优化超参数**：调整学习率（推荐3e-4）、EMA衰减率（推荐0.9999）、batch size
4. **充分利用概率信息**：重点展示CRPS指标和预测区间，突出扩散模型在不确定性估计方面的优势
