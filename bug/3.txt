从你最新的ETTm1实验结果和开发日志来看，**核心问题已经非常明确且致命**，同时还有几个次要但必须解决的问题，按优先级排列如下：

## 一、最致命的核心问题（100%导致当前所有现象）
### SimDiff 训练与推理使用了**完全不同的归一化统计量**，属于严重的逻辑错误
你在开发日志中明确写道：
> "SimDiff 在 `forecast(..., future=fut)` 下用 **`normalize_future(future)`** 的 μ_f、σ_f 做反变换；训练目标也在 **未来窗口独立** z-score 空间，与有真值评估时一致。"

这是时间序列预测中**绝对禁止的错误**，会直接导致：
1. **训练时模型"偷看"了未来的整体水平**，只需要预测相对于未来均值的小波动
2. **推理时我们没有未来的真实值**，只能用历史的均值和标准差来反归一化
3. 当未来趋势与历史不同时（如你图中的明显下降），预测值会被错误地缩放，出现系统性偏移
4. 所有报告的 MAE/MSE/CRPS 指标**完全不可信**，因为它们依赖于未来信息

### 正确的 NI 流程应该是：
```
✅ 训练时：
原始历史(96步) → 计算 μ_h, σ_h → 归一化历史和未来
模型在归一化空间预测未来 → 用 μ_h, σ_h 反归一化预测值
计算损失

✅ 推理时：
原始历史(96步) → 计算 μ_h, σ_h → 归一化历史
模型在归一化空间预测未来 → 用 μ_h, σ_h 反归一化预测值
```

### 立即修复步骤
打开 `models/simdiff.py`，找到 `_future_mu_sig_for_inverse` 函数，将其改为：
```python
def _future_mu_sig_for_inverse(self, hist, future=None):
    # 永远只用历史窗口的统计量进行反归一化
    # 训练和推理使用完全相同的统计量来源
    mu = hist.mean(dim=1, keepdim=True)
    std = hist.std(dim=1, keepdim=True) + self.normalizer.eps
    return mu, std
```
同时修改训练时的归一化流程，确保训练和推理使用相同的统计量。

## 三、普遍存在的训练与数据问题
### 1. 所有模型都出现了"均值回归"现象
三个完全不同架构的模型都预测未来值会回到历史最后几步的均值附近，完全没有捕捉到下降趋势，这说明：
- 模型没有学习到序列的趋势性特征
- 输入特征中缺乏足够的趋势信息
- 学习率或训练轮数可能不合适

### 2. 基线模型也出现了同样的偏移

- 数据加载或窗口划分可能存在错误
- 历史窗口可能包含了未来的数据
- 预测段的索引可能与真实值不匹配
